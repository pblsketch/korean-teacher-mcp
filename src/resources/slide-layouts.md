# 슬라이드 레이아웃 가이드 (교육용 7종)

> export_pptx 도구의 `<!-- layout:... -->` 힌트 주석으로 지정하는 슬라이드 레이아웃 가이드입니다.
> 교육 자료 작성 맥락에 최적화된 7종을 선별했습니다.
> PptxGenJS 좌표계(16:9 = 10" × 5.625" 기준) 기반.

---

## 레이아웃 힌트 문법

마크다운 슬라이드(H1/H2) 바로 아래에 HTML 주석으로 지정:

```markdown
# 단원 학습 목표

<!-- layout:two-column -->

- 작품의 주제 파악
- 인물의 심리 분석
- 문학적 표현 이해
- 시대 배경 이해
- 작가의 의도 파악
- 현대적 의의 탐색
```

힌트가 없으면 기본 `title-content` 레이아웃으로 렌더링됩니다 (기존 동작).

---

## 1. Title Slide (타이틀)

**힌트**: `<!-- layout:title -->` (또는 기존 `title` 파라미터)

**용도**: 단원 소개, 수업 시작

```
┌──────────────────────────────────────┐
│                                      │
│                                      │
│         [큰 제목 — 36pt 볼드]         │
│                                      │
│         [부제목 — 18pt]               │
│                                      │
│                                      │
└──────────────────────────────────────┘
```

**좌표 (PptxGenJS)**
- 제목: `{ x: 0.5, y: 1.5, w: 9, h: 1.5 }`
- 부제목: `{ x: 0.5, y: 3.2, w: 9, h: 1.0 }`

**사용 예**: 단원 첫 슬라이드, 연수 오프닝

---

## 2. Title + Content (제목+내용) — **기본값**

**힌트**: `<!-- layout:title-content -->` (생략 가능 — 힌트 없으면 이것)

**용도**: 일반 설명 슬라이드

```
┌──────────────────────────────────────┐
│ ████ [타이틀 바 24pt] ████████████  │
├──────────────────────────────────────┤
│                                      │
│  • 불릿 1                             │
│  • 불릿 2                             │
│  • 불릿 3                             │
│  • 불릿 4                             │
│                                      │
└──────────────────────────────────────┘
```

**좌표**
- 타이틀 바: `{ x: 0, y: 0, w: 10, h: 1.2 }`
- 본문: `{ x: 0.5, y: 1.5, w: 9, h: 4.0 }`

---

## 3. Two Column (2단)

**힌트**: `<!-- layout:two-column -->`

**용도**: 불릿 6개 이상, 좌우 분할 설명, 전/후 비교

```
┌──────────────────────────────────────┐
│ ████ [타이틀 바] ████████████████   │
├──────────────────┬───────────────────┤
│                  │                   │
│  • 좌측 불릿 1    │  • 우측 불릿 1     │
│  • 좌측 불릿 2    │  • 우측 불릿 2     │
│  • 좌측 불릿 3    │  • 우측 불릿 3     │
│                  │                   │
└──────────────────┴───────────────────┘
```

**좌표**
- 좌측 열: `{ x: 0.5, y: 1.5, w: 4.4, h: 3.8 }`
- 구분선: `{ x: 5.0, y: 1.5, w: 0, h: 3.8 }` (세로)
- 우측 열: `{ x: 5.2, y: 1.5, w: 4.4, h: 3.8 }`

**분할 규칙**: 불릿 리스트를 반으로 나누어 좌/우 배치.

---

## 4. Comparison (비교)

**힌트**: `<!-- layout:comparison -->`

**용도**: 명시적 대조 (A vs B, 장점/단점, 개념 비교)

```
┌──────────────────────────────────────┐
│ ████ [타이틀 바] ████████████████   │
├──────────────────┬───────────────────┤
│ ┌──────────────┐ │ ┌──────────────┐ │
│ │  A 헤더 (파랑)│ │ │  B 헤더 (빨강)│ │
│ ├──────────────┤ │ ├──────────────┤ │
│ │              │ │ │              │ │
│ │  A 내용       │ │ │  B 내용       │ │
│ │              │ │ │              │ │
│ └──────────────┘ │ └──────────────┘ │
└──────────────────┴───────────────────┘
```

**좌표**
- A 헤더: `{ x: 0.5, y: 1.5, w: 4.4, h: 0.6 }` (fill: accent 파랑)
- A 내용 박스: `{ x: 0.5, y: 2.1, w: 4.4, h: 3.2 }` (fill: #F8FAFD)
- B 헤더: `{ x: 5.1, y: 1.5, w: 4.4, h: 0.6 }` (fill: 대조색 e.g. E74C3C)
- B 내용 박스: `{ x: 5.1, y: 2.1, w: 4.4, h: 3.2 }` (fill: #FDF8F8)

**입력 규칙**: 불릿 중 "A:" 또는 "vs" 뒤는 B로 분할.

---

## 5. Statistics (통계)

**힌트**: `<!-- layout:statistics -->`

**용도**: 큰 숫자 강조 (달성도, 성취 비율, 설문 결과)

```
┌──────────────────────────────────────┐
│ ████ [타이틀 바] ████████████████   │
├──────────────────────────────────────┤
│                                      │
│           ╔════════╗                 │
│           ║   85%  ║  ← 100pt        │
│           ╚════════╝                 │
│                                      │
│       [설명 문장 — 16pt]              │
│                                      │
└──────────────────────────────────────┘
```

**좌표**
- 큰 숫자: `{ x: 2, y: 1.8, w: 6, h: 2, fontSize: 80, bold: true, align: 'center' }`
- 설명: `{ x: 1, y: 4.0, w: 8, h: 1.0, fontSize: 16, align: 'center' }`

**입력 규칙**: 불릿 첫 줄이 숫자+단위(`%`, `명`, `점` 등) 포함 → 큰 숫자. 나머지 → 설명.

---

## 6. Quote (인용)

**힌트**: `<!-- layout:quote -->`

**용도**: 작가 인용, 핵심 문장, 학생 발언

```
┌──────────────────────────────────────┐
│ ████ [타이틀 바] ████████████████   │
├──────────────────────────────────────┤
│                                      │
│   ▌                                  │
│   ▌ "인용 문장                        │
│   ▌  여러 줄 가능"                    │
│   ▌                                  │
│                                      │
│               — 작가명                │
│                                      │
└──────────────────────────────────────┘
```

**좌표**
- 좌측 세로 바: `{ x: 0.8, y: 1.8, w: 0.1, h: 2.5, fill: accent }`
- 인용문: `{ x: 1.2, y: 1.8, w: 8, h: 2.5, fontSize: 20, italic: true }`
- 출처: `{ x: 1.2, y: 4.5, w: 8, h: 0.5, fontSize: 14, align: 'right' }`

**입력 규칙**: 불릿 중 `"..."` 로 감싼 것은 인용, `— 이름` 또는 `-- 이름`은 출처.

---

## 7. Process (프로세스)

**힌트**: `<!-- layout:process -->`

**용도**: 글쓰기 과정, 탐구 절차, 시간 순 진행

```
┌──────────────────────────────────────┐
│ ████ [타이틀 바] ████████████████   │
├──────────────────────────────────────┤
│                                      │
│  ┌───┐    ┌───┐    ┌───┐    ┌───┐   │
│  │ 1 │──▶│ 2 │──▶│ 3 │──▶│ 4 │   │
│  └───┘    └───┘    └───┘    └───┘   │
│   관찰     가설     실험     결론    │
│                                      │
└──────────────────────────────────────┘
```

**좌표**
- 원/사각형 (4개 균등): x = 0.8, 2.8, 4.8, 6.8 / y = 2.2, w/h = 1.2
- 화살표: 각 원 사이
- 라벨: 각 원 아래 `y: 3.5, h: 0.4`

**입력 규칙**: 번호 목록 (`1. ... 2. ... 3. ...`) 감지 시 각 항목을 단계로 변환. 최대 5단계.

---

## 8. Section Header (섹션 구분)

**힌트**: `<!-- layout:section-header -->`

**용도**: 단원 전환, 챕터 구분

```
┌──────────────────────────────────────┐
│████████████████████████████████████│
│████████████████████████████████████│
│████    [큰 제목 — 흰색]    ████████│
│████    [부제목 — 연한색]    ██████│
│████████████████████████████████████│
│████████████████████████████████████│
└──────────────────────────────────────┘
```

**좌표**
- 전체 배경: titleBg 색상
- 제목: `{ x: 0.5, y: 2.0, w: 9, h: 1.0, fontSize: 44, bold: true, color: white, align: center }`
- 부제목: `{ x: 0.5, y: 3.2, w: 9, h: 0.6, fontSize: 16, color: 'CCCCCC', align: center }`

**입력 규칙**: H1/H2 제목만 있고 불릿이 없을 때 자동 적용 가능 (향후). 현재는 명시적 힌트만.

---

## 9. Closing (마무리)

**힌트**: `<!-- layout:closing -->`

**용도**: 정리, Q&A, 감사 인사

```
┌──────────────────────────────────────┐
│                                      │
│                                      │
│         [감사합니다 — 크게]           │
│                                      │
│         [Q&A 또는 연락처]             │
│                                      │
└──────────────────────────────────────┘
```

Title Slide와 유사하나 배경은 `titleBg` 대신 `contentBg`. 실무상 사용 빈도 낮아 우선순위 낮음.

---

## 입력 → 레이아웃 매핑 (힌트 기반)

export_pptx는 **자동 추정을 하지 않습니다**. 오로지 명시적 힌트 주석으로만 레이아웃을 선택합니다:

| 힌트 | 렌더링 함수 | 최소 콘텐츠 요구 |
|------|-----------|--------------|
| (없음) | `title-content` | 제목 필수, 불릿·표 선택 |
| `<!-- layout:two-column -->` | `two-column` | 불릿 2개 이상 |
| `<!-- layout:comparison -->` | `comparison` | 불릿 2개 이상 (A:/B: 또는 좌우 분할) |
| `<!-- layout:statistics -->` | `statistics` | 불릿 1개 이상 (첫 줄이 숫자) |
| `<!-- layout:quote -->` | `quote` | 불릿 중 "..." 인용구 |
| `<!-- layout:process -->` | `process` | 번호 목록 2개 이상 |
| `<!-- layout:section-header -->` | `section-header` | 제목만 |
| `<!-- layout:title -->` | `title-slide` | 제목+부제목 |

힌트가 잘못됐거나 최소 콘텐츠를 충족 못 하면 → **자동으로 `title-content`로 폴백** (경고 없이 조용히 처리).

## 12 디자인 스타일별 호환 레이아웃

각 스타일에 특히 잘 어울리는 레이아웃 3개씩:

| 스타일 | 권장 레이아웃 |
|--------|----------|
| Dark Academia | title, quote, section-header |
| Swiss International | title-content, comparison, statistics |
| Nordic Minimalism | title, quote, two-column |
| Bento Grid | two-column, comparison, process |
| Pastel Soft UI | title-content, statistics, closing |
| Gradient Mesh | title, section-header, quote |
| Glassmorphism | comparison, statistics, process |
| Architectural Blueprint | process, comparison, statistics |
| Monochrome Minimal | quote, title, statistics |
| Editorial Magazine | quote, two-column, title |
| Typographic Bold | statistics, title, section-header |
| Duotone Color Split | comparison, two-column, statistics |

## 주의 사항

1. **힌트는 H1/H2 다음 줄**에 배치. 슬라이드 중간에 있으면 무시됨.
2. **한 슬라이드 한 레이아웃**. 한 슬라이드에 여러 힌트 작성 시 첫 번째만 적용.
3. **표와 레이아웃 충돌**: 표가 있는 슬라이드는 레이아웃 힌트가 있어도 표 우선. 표 데이터를 차트로 바꾸려면 `<!-- chart:... -->` 힌트 사용.
4. **힌트 오타**: 알 수 없는 힌트 값이면 조용히 `title-content`로 폴백. 경고 없음.
