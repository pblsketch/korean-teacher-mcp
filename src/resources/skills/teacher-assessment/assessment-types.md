# 평가 유형별 가이드

> korean-teacher-mcp | teacher-assessment 참조 가이드

평가 개발 프롬프트 가이드 — 형성평가, 총괄평가, 수행평가, 루브릭 개발을 위한 프롬프트 유형별 정리입니다.

## 평가 프롬프트의 PARTS 적용

| 요소 | 평가 맥락에서의 적용 |
|------|---------------------|
| **P: Persona** | 평가 설계 전문가, 평가 문항 개발자, 루브릭 설계자 |
| **A: Act** | 생성하다, 평가하다, 분석하다, 개별화하다 |
| **R: Recipient** | 특정 학년/수준의 학생, 평가 목적 |
| **T: Theme** | 평가할 개념/기능, 성취기준 |
| **S: Structure** | 문항 유형, 루브릭 형식, Bloom 수준 |

## 한국 교육 맥락의 평가 체계

| 평가 유형 | 비중 | 특성 | 프롬프트 강조점 |
|----------|------|------|----------------|
| **지필평가** | 40-60% | 선택형+서술형 | 객관적 채점 기준 |
| **수행평가** | 40-60% | 과정 중심 | 과정 평가, 피드백 |
| **형성평가** | 비반영 | 학습 점검 | 즉각적 피드백 |

## 2.1 퀴즈 및 문항 생성 (Quiz Generation)

보기형 문항은 <보기> 자료(예문, 대화, 표, 지문 등)를 제시하고 이를 분석·적용·판단하게 하는 유형입니다.

- 20문항 기준: 선택형 10 + 보기형 4 + 서술형 6
- 난이도 배분: 하 6 / 중 8 / 상 6
- 보기형은 중~상 난이도로 배치하며, 실제 텍스트나 예문을 활용합니다

**활용 팁**
- "이해도 점검"을 강조하면 추론과 적용 수준의 문항이 포함됩니다
- 정답 해설에 "학생 친화적 설명"을 요청하면 학생 자기 점검에 활용 가능합니다

**적합한 상황**
- 수업 중 즉각적인 학습 확인이 필요할 때
- 단원 마무리 전 학생 이해도 파악이 필요할 때

## 2.2 수행과제 설계 (Performance Task)

수행과제 프롬프트에는 다음을 포함합니다:
- 학생용 과제 안내문
- 평가 기준표 (루브릭)
- 과정 평가 체크포인트
- 우수 예시 설명

**활용 팁**
- "과정 중심 평가"를 강조하면 중간 점검 활동이 포함됩니다
- "학생 자기평가"를 요청하면 메타인지 발달을 위한 도구가 포함됩니다

## 2.3 루브릭 개발 (Rubric Design)

분석적 루브릭 개발 시 포함 요소:
- 영역별 상세 평가 기준
- 수준별 구체적 행동 지표
- 각 수준의 예시 설명
- 학생 자기평가용 간소화 버전

## 2.4 기존 평가 변형 (Assessment Differentiation)

기존 평가를 동일한 학습 목표를 유지하면서 대안적 버전으로 재구성합니다.
- 원본 평가
- 변형 목적 (재시험/보충 학습/수준별 평가)
- 유지 사항 (성취기준, 문항 수, 평가 시간)
- 변경 사항 (난이도, 적용 맥락)

## 2.5 데이터 기반 수업 개선 (Data Analysis for Instruction)

형성평가 결과 분석 항목:
1. 전체 학습 도달도 분석
2. 문항별 정답률과 오류 패턴
3. 공통 오개념 식별
4. 재지도가 필요한 핵심 개념
5. 수준별 소그룹 구성 제안
6. 후속 수업 활동 3가지 제안

## 2.6 사례 기반 평가 설계 (Case-Based Assessment)

실제 상황 기반 평가 포함 요소:
1. 실제 교실 상황을 반영한 사례 시나리오 (500자 내외)
2. 분석 질문 3개 (이론 적용, 대안 제안, 자기 성찰)
3. 루브릭 (증거 활용, 개념 정확성 기준 포함)
4. 학생 자기평가용 안내문

## 프롬프트 품질 체크리스트

- [ ] 평가 목적(형성/총괄/진단)이 명확한가?
- [ ] 관련 성취기준이 포함되어 있는가?
- [ ] 문항 유형과 수가 명시되어 있는가?
- [ ] 채점 기준(루브릭/정답 해설)이 요청되어 있는가?
- [ ] 인지 수준이 적절하게 분배되어 있는가?
- [ ] 한국 평가 맥락(지필+수행 병행)이 반영되어 있는가?

## 후속 프롬프트 제안

1. **수준별 변형**: "이 평가를 기초/심화 수준으로 변형해주세요"
2. **피드백 설계**: "이 평가 결과에 대한 학생별 피드백 템플릿을 만들어주세요"
3. **재지도 계획**: "이 평가에서 오류가 많은 문항에 대한 재지도 활동을 제안해주세요"
4. **자기평가 도구**: "학생 자기평가용 체크리스트를 만들어주세요"
