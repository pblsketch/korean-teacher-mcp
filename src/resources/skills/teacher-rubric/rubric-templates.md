# 루브릭 JSON 출력 템플릿

> korean-teacher-mcp | teacher-rubric 참조 가이드

루브릭 응답은 반드시 다음 JSON 형식으로 작성합니다.

## 응답 형식 (JSON 스키마)

```json
{
  "thinking": "평가 준거 도출 과정에 대한 상세한 설명. 어떤 근거로 이러한 평가 준거를 선정했는지 설명합니다.",
  "criteria": [
    {
      "name": "준거명",
      "description": "준거에 대한 상세한 설명. 이 준거의 의미와 중요성을 설명합니다.",
      "rationale": "준거 선정 이유. 왜 이 준거가 이 프로젝트에서 중요한지 설명합니다.",
      "levels": [
        {
          "levelName": "수준명",
          "description": "이 수준에서 학생 작품이 어떻게 보이는지 구체적이고 관찰 가능한 특성으로 서술합니다."
        }
      ]
    }
  ],
  "usageSuggestions": {
    "teacher": {
      "before": ["수업 전 활용 방법 1", "수업 전 활용 방법 2", "..."],
      "during": ["수업 중 활용 방법 1", "수업 중 활용 방법 2", "..."],
      "after": ["수업 후 활용 방법 1", "수업 후 활용 방법 2", "..."]
    },
    "student": {
      "planning": ["학습 계획 단계 활용법 1", "학습 계획 단계 활용법 2", "..."],
      "process": ["학습 과정 중 활용법 1", "학습 과정 중 활용법 2", "..."],
      "reflection": ["학습 완료 후 활용법 1", "학습 완료 후 활용법 2", "..."]
    }
  }
}
```

## 주의사항

- 평가 준거는 **항상 5개**를 작성합니다.
- 각 평가 준거는 성취기준의 핵심 문장을 근거로 도출하며, 준거 설명(rationale)에 어떤 성취기준 문장을 반영했는지 명시합니다.
- 수행 수준은 사용자가 요청한 개수(3~5단계)에 맞춰 작성합니다.
- 각 준거의 수준은 평행 구조를 유지하며, 같은 측면을 다른 질적 수준으로 기술합니다.
- 학생 친화적인 언어를 사용하되, 명확하고 구체적으로 작성합니다.
- 단순 계수나 개수가 아닌 **질적 특성**을 기술합니다.

## 교과별 준거 선정 참고

### 국어
- 주제 이해의 깊이 · 표현의 적절성 · 논리 구조 · 근거의 타당성 · 언어 규범 준수

### 수학
- 개념 이해 · 문제 해결 전략 · 계산 정확성 · 논리적 추론 · 의사소통(풀이 과정 서술)

### 과학
- 탐구 설계 · 자료 수집과 분석 · 개념 적용 · 결과 해석 · 과학적 의사소통

### 사회
- 개념 이해 · 자료 분석 · 다양한 관점 이해 · 논리적 주장 구성 · 실제 적용

각 교과의 준거는 해당 교육과정 성취기준과 1:1로 정렬되어야 하며, 성취기준에 명시되지 않은 일반 역량(협력·창의성 등)은 독립 준거로 사용하지 않습니다.

## 교사·학생 활용 제안 작성 가이드

**교사(teacher)**
- before: 수업 계획·과제 안내·학생 사전 안내
- during: 수업 중 형성 피드백·진행 점검·소그룹 지도
- after: 총평·성적 산출·차시 계획 반영

**학생(student)**
- planning: 과제 이해·자기 목표 설정·계획 수립
- process: 진행 중 자기점검·동료 피드백·수정 방향
- reflection: 완성 후 자기평가·성장 기록·차기 목표
