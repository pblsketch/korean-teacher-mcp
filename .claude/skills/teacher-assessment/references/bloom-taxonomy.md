# Bloom 인지 수준별 문항 설계

> korean-teacher-mcp | teacher-assessment 참조 가이드

프롬프트에 다음 수준을 명시하면 해당 수준의 문항이 설계됩니다.

## Bloom 수준별 동사와 예시 키워드

| 수준 | 동사 | 평가 문항 예시 키워드 |
|------|------|---------------------|
| **기억(Remember)** | 정의하다, 나열하다 | "~은 무엇인가?", "나열하시오" |
| **이해(Understand)** | 설명하다, 요약하다 | "설명하시오", "비교하시오" |
| **적용(Apply)** | 적용하다, 계산하다 | "~를 활용하여", "해결하시오" |
| **분석(Analyze)** | 구분하다, 분석하다 | "원인을 분석하시오", "차이점은?" |
| **평가(Evaluate)** | 평가하다, 비판하다 | "타당성을 평가하시오" |
| **창안(Create)** | 설계하다, 제안하다 | "새로운 방안을 제안하시오" |

## 수준 혼합 원칙

- 형성평가: 기억·이해·적용 중심 (하위 3단계)
- 총괄평가: 6단계 전체 균형 분배
- 수행평가: 적용·분석·평가·창안 중심 (상위 4단계)

## 인지 수준별 문항 활용 팁

- 단순 기억 문항은 20% 이내로 제한하여 고차 사고 유도
- "이해도 점검"을 강조하면 추론과 적용 수준의 문항이 자연스럽게 포함됩니다
- 프롬프트에 Bloom 수준을 명시적으로 요청하면 문항 분배가 균형 있게 조정됩니다
